2) Saudi procurement compliance rules (imp lementable 
rule set)  
Below is a rules engine spec  your devs can implement (flags, severities, and required 
evidence).  
2.1 Baseline Saudi Compliance (Always On)  
Rule SA -VAT -01 — Supplier VAT / ZATCA Verification  
 Check:  Supplier has VAT registration number ( if required by engagement) and 
provides VAT -compliant invoice capability.  
 Evidence:  VAT number, tax certificate, or verified vendor master record.  
 Severity:  HIGH  
 Action:  Block “Approved Supplier” status until verified.  
Rule SA -VAT -02 — VAT Rate & Tax Treat ment Consistency  
 Check:  Quote shows VAT treatment correctly (VAT -exclusive vs inclusive) and does 
not apply inconsistent tax lines.  
 Evidence:  Quotation lines + terms.  
 Severity:  MEDIUM  
 Action:  Flag for correction before final selection.  
Rule SA -PO-01 — No PO Without Approval  
 Check:  PO creation only after approvals recorded (Procurement + Commercial + 
Project Director where applicable).  
 Evidence:  Approval log record ids.  
 Severity:  HIGH  
 Action:  Hard block.  
Rule SA -AML -01 — Supplier Identity  & Banking Controls (Basic)  
 Check:  Supplier legal name matches bank account name; prevent invoice fraud.  
 Evidence:  Vendor master + bank letter/IBAN verification.  
 Severity:  HIGH  
 Action:  Block payment handover until verified.  
2.2 Localization / Saudi -specifi c Project Requirements (Configurable)  
Rule SA -LOC -01 — Local Content Requirement  
 Check:  If project requires Saudi/local content, supplier meets required thresholds or 
provides compliant subcontracting plan.  
 Evidence:  Local content declaration.  
 Severity:  HIGH  
 Action:  Block shortlist unless exception approved.  
Rule SA -CERT -01 — Mandatory Certifications  
 Check:  Supplier holds required certifications (ISO, safety, trade licenses, client -
specific).  
 Evidence:  Certificates with expiry dates.  
 Severity:  HIGH  
 Action : Block shortlist if missing/expired.  
2.3 Owner/Authority Specific Rule Packs (Enable per project)  
Create toggles per project for:  
 Aramco Pack  (vendor approval, Aramco specs, documentation formatting)  
 SEC Pack  (electrical compliance, approved material list s) 
 RCJY Pack  (Royal Commission safety & documentation)  
 NEOM / Red Sea / Diriyah Pack  (their procurement templates, ESG, reporting)  
Each pack should include:  
 Required templates  
 Required approvals  
 Required supplier status (e.g., “approved vendor list only”)  
 Document naming conventions and submission formats  
2.4 Scope & Commercial Integrity (Always On)  
Rule SA -SCOPE -01 — Quote vs Scope Deviation  
 Check:  Quote missing any BOQ line, excludes mandatory scope, or changes spec.  
 Evidence:  Normalized quote comparison output.  
 Severity:  HIGH  
 Action:  Flag and require clarification / revised quote.  
Rule SA -COMM -01 — Payment Terms Deviation  
 Check:  Payment terms deviate from project procurement policy.  
 Severity:  MEDIUM/HIGH (config)  
 Action:  Route to commercial for explicit a pproval.  
Rule SA -TIME -01 — Lead Time vs Program Risk  
 Check:  Lead time exceeds project milestone needs.  
 Severity:  MEDIUM  
 Action:  Flag risk; suggest alternate suppliers.

3⃣ SAUDI PROCUREMENT 
COMPLIANCE RULE ENGINE  
(Deterministic pseudo -code)  
function run_procurement_compliance(quote, supplier, project):  
 
  flags = []  
 
  // VAT / ZATCA  
  if supplier.vat_required  and supplier.vat_number is null:  
      flags.add(HIGH, "SA -VAT-01", "Supplier VAT not verified")  
 
  if quote.vat_rate != 0.15 and quote.vat_applicable:  
      flags.add(MEDIUM, "SA -VAT-02", "Incorrect VAT rate")  
 
  // Localization  
  if project.requires_local_content:  
      if supplier.local_content_pct < project.min_local_content:  
          flags.add(HIGH, "SA -LOC-01", "Local content requirement not met")  
 
  // Certifications  
  for cert in project.required_certifications:  
      if cert not in supplier.certificates or cert.expired:  
          flags.add(HIGH, "SA -CERT-01", "Missing or expired certification")  
 
  // Scope integrity  
  for boq_line in project.boq:  
      if boq_line not in quote.items:  
          flags.add(HIGH, "SA -SCOPE-01", "BOQ line missing in quote")  
 
  // Payment terms  
  if quote.payment_terms != project.allowed_payment_terms:  
      flags.add(MEDIUM, "SA -COMM-01", "Payment terms deviation")  
 
  // Decision  
  if any flags.severity == HIGH:  
      status = "BLOCKED"  
  else: 
      status = "REVIEW"  
 
  return {status, flags}
